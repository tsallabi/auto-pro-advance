### المرحلة 1: التسجيل المتقدم + KYC 🔐

**الملفات:** `AuthModal.tsx`, `server.ts`, `types.ts`

#### Frontend (AuthModal.tsx)

- [ ] إضافة خطوة ثانية بعد البيانات الأساسية تشمل:

- [ ] تحديث `/api/auth/register` ليقبل الحقول الجديدة
- [ ] حالة المستخدم الجديد = `pending_approval`

### المرحلة 2: موافقة المدير على المشتري ✅

**الملفات:** `AdminDashboard.tsx`, `server.ts`

- [ ] قسم جديد "طلبات الانضمام" في القائمة الجانبية

- [ ] `GET /api/admin/pending-users` - جلب المستخدمين المعلقين

### المرحلة 5: نتيجة المزاد (فوز / خسارة) 🏆

**الملفات:** `server.ts`, `StoreContext.tsx`, `UserDashboard.tsx`

#### عند الفوز

- [ ] إشعار فوري: "تهانينا! فزت بسيارة [اسم] بمبلغ $[X]"
- [ ] إنشاء 3 فواتير تلقائياً:
  1. **فاتورة الشراء** (purchase) - قيمة المزايدة + عمولة المنصة
  2. **فاتورة النقل** (transport) - نقل داخلي (معلقة حتى دفع فاتورة الشراء)
  3. **فاتورة الشحن** (shipping) - شحن دولي (معلقة حتى وصول المستودع)
- [ ] نقل السيارة لقسم "سياراتي المربوحة"
- [ ] حالة الشحن تبدأ: "بانتظار الدفع"

#### عند الخسارة

- [ ] إشعار: "للأسف لم تفز بسيارة [اسم]. المبلغ المحجوز تم تحريره."
- [ ] تحرير المبلغ المحجوز من القوة الشرائية
- [ ] اقتراح سيارات مشابهة

---

### المرحلة 6: نظام الفواتير المتعددة 📄

**الملفات:** `UserDashboard.tsx` (invoices), `server.ts`

#### نظام 3 فواتير لكل سيارة

| # | النوع | الحالة الافتراضية | الشرط |
|---|---|---|---|
| 1 | `purchase` قيمة الشراء | `unpaid` | تُنشأ فوراً بعد الفوز |
| 2 | `transport` النقل الداخلي | `pending` (معلقة) | تُفعّل بعد دفع فاتورة الشراء |
| 3 | `shipping` الشحن الدولي | `pending` (معلقة) | تُفعّل بعد وصول السيارة للمستودع |

#### واجهة الفواتير

- [ ] عرض الفواتير مجمّعة حسب السيارة
- [ ] كل سيارة تعرض فواتيرها الثلاثة كـ Timeline
- [ ] لا يمكن دفع فاتورة النقل قبل دفع فاتورة الشراء
- [ ] لا يمكن دفع فاتورة الشحن قبل وصول المستودع
- [ ] كود الاستلام (Pickup Auth) يظهر بعد دفع فاتورة الشراء

### المرحلة 7: التتبع والشحن 🚚

**الملفات:** `UserDashboard.tsx` (قسم جديد: التتبع), `server.ts`

#### حالات الشحن

```
بانتظار الدفع → تم الدفع → قيد النقل → في المستودع → جاري الشحن → التخليص الجمركي → تم التوصيل
```

| الحالة (Status Key) | بالعربية | الأيقونة |
|---|---|---|
| `awaiting_payment` | بانتظار الدفع | 💳 |
| `paid` | تم الدفع | ✅ |
| `in_transport` | قيد النقل | 🚛 |
| `in_warehouse` | في المستودع | 🏭 |
| `in_shipping` | جاري الشحن | 🚢 |
| `customs` | التخليص الجمركي | 📋 |
| `delivered` | تم التوصيل | 🎉 |

#### واجهة التتبع

- [ ] Timeline عمودي مع تأثيرات بصرية
- [ ] الخطوة الحالية مُضاءة والسابقة مُكتملة والقادمة رمادية
- [ ] توقيت تقديري لكل مرحلة
- [ ] زر "تحديث الحالة" (للمدير فقط)
